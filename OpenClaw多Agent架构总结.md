# OpenClaw 多 Agent 架构总结

## 🦞 OpenClaw Agent 架构

从配置文件看，OpenClaw 支持：

```json
" agents": {
  "defaults": {
    "model": { "primary": "minimax/MiniMax-M2.7" }
  }
}
```

### 单机可以运行多少？

| 资源 | 限制因素 |
|------|---------|
| **内存** | 每个 Agent 约 100-500MB |
| **CPU** | 并行数量取决于核心数 |
| **端口** | 每个 Gateway 占用一个端口 |

## 🎯 总指挥模式需求

用户需求：层级 Agent 架构

```
用户 → 总指挥 → 执行 Agent 1
                   → 执行 Agent 2
                   → 执行 Agent 3
```

---

## 🚀 三种实现方案

### 方案1：你一个人做所有事

```
你 → 你自己 → 写代码 + 查资料 + 调试
```

**原理**：同一台电脑、同一个人、不同的"思考模式"来回切换

- 你说"帮我 TDD 开发"，我就切换成 TDD 模式
- 你说"帮我调试"，我就切换成调试模式
- 本质上还是**同一个大脑**在干活

**像什么**：你一个人在家，一会儿当厨师做饭，一会儿当清洁工打扫，一会儿又当程序员写代码

---

### 方案2：同一个人，但固定角色

```
你 → 固定角色"总指挥" → 分派任务
```

**原理**：给同一个人设定一个**永远不变的角色**

- 我永远是"总指挥"角色
- 我只负责思考和规划
- 具体执行让别人干

**像什么**：你雇了一个人当管家，他只动嘴不动手，脏活累活都指挥别人干

---

### 方案3：真的雇多个人

```
你 → 管家A → 指挥 → 厨师B
                      → 清洁工C
                      → 司机D
```

**原理**：**每个 Agent 是独立运行的程序**，有自己独立的内存和进程

- 管家A在 port 18789 运行
- 厨师B在 port 18790 运行
- 他们**同时存在，同时工作**

**像什么**：真的雇了4个员工，每个都在上班，你跟管家说话，他去指挥别人

---

## ⚖️ 对比总结

| | 方案1 | 方案2 | 方案3 |
|---|---|---|---|
| **人数** | 1人 | 1人 | 4人 |
| **同时干活** | ❌ 切换模式 | ❌ 切换模式 | ✅ 同时干 |
| **内存占用** | 最低 | 低 | 高 |
| **Token 消耗** | 1份 | 1份 | 4份 |
| **复杂度** | 简单 | 中等 | 复杂 |

## 🎯 适合你的场景

**目标**：消耗 Token（600次/5小时）

| 方案 | Token 消耗 | 推荐度 |
|------|-----------|--------|
| 方案1 | 1份 | ⭐⭐⭐⭐⭐ |
| 方案2 | 1份 | ⭐⭐⭐⭐ |

## 🔧 技术实现方式

### 方案1 & 2：单 Agent 模式
- 使用 `Skill` 工具调用不同的技能模块
- Agent 根据上下文自动切换模式
- 所有任务在同一进程中执行

### 方案3：多 Agent 模式
- 启动多个 Gateway 实例
- 每个实例占用独立端口
- 通过 RPC 或 HTTP 通信

## 🍎 Mac 配置分析

### 2026-03-25 实测

| 项目 | 配置 |
|------|------|

| **芯片** | Apple M1 |

### M1 + 16GB 能同时跑多少 Agent？

| 方案 | Agent 数量 | 内存预估 | CPU |
|------|-----------|---------|-----|
| **方案1** | 1 个 | ~1GB | 低 |
| **方案2** | 1 个 | ~1GB | 低 |
| **方案3** | 5-10 个 | ~5-10GB | 中 |

### 结论

| 方案 | Token 消耗 | 推荐度 |
|------|-----------|--------|
| **方案1** | ⭐ (1份) | ⭐⭐⭐⭐⭐ **最推荐** |
| 方案2 | ⭐ (1份) | ⭐⭐⭐⭐ |
| 方案3 | ⭐⭐⭐⭐ (4份) | ⭐ (浪费) |

**您的 Mac 完全能跑方案3**，但从消耗 Token 角度来说：

> **方案1 最划算** = 只用一个 Agent 反复干活，把 600 次额度用完

**16GB 内存表现**：
- 方案1：轻松跑，还有 15GB 空闲
- 方案3：跑 10 个也绰绰有余

**但问题是**：您有 600 次/5小时，用 1 个 Agent 足够了，多了反而 Token 不够分。
